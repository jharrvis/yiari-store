# Documentation Structure

## Folders
- `plan/`: active implementation plans and future change roadmaps.
- `docs/reports/`: historical fix reports, migration summaries, and verification artifacts.
- `docs/notes/`: reserved for temporary technical notes or audit notes during development.

## Current Plan
- `plan/plugin-update-plan-2026-07-10.md`: plan for refactoring product handling, Midtrans-only payment flow, KiriminAja shipping, and fulfillment automation.

## Notes
- Keep `AGENTS.md`, `CHANGELOG.md`, and `readme.txt` in the repository root.
- New status reports should go to `docs/reports/` unless they are active execution plans.
